Three approaches for the Store Poster feature

Poster opens when you click a "Poster" button on an Amber/Red store chip in the sidebar.

A

Inline slide-in panel (Recommended)

Clicking "Poster" on a store chip slides open a full-width poster panel inside the main content area — right below the sidebar chip. The poster uses a gradient background, bold discount headline, catchy tagline from built-in templates, and store name. A "Print / Save" button triggers browser print. No modal, no overlay — stays inside the page flow.

Pros

  • Seamless UX — no modal z-index issues
  • Easy to compare posters side-by-side
  • Simple to implement

Cons

  • Takes up vertical space in the sidebar
B

Full-screen overlay modal

Clicking "Poster" opens a centered full-screen modal with the poster displayed large and prominent. A dark backdrop dims the rest of the page. The poster uses cinematic gradient, huge discount %, and store branding. Close button or Escape to dismiss. Print button inside the modal.

Pros

  • Most poster-like feel — dramatic, full attention
  • Ideal for demo / presentation

Cons

  • More code (backdrop, focus trap, keyboard close)
C

Dedicated poster section in main content

A "Store Posters" section auto-appears in the main content area whenever Amber/Red stores are present. Individual store posters render as horizontal cards — no extra click on the chip needed. Each card has store name, avg discount, tier badge, and tagline.

Pros

  • Always visible — no extra interaction needed
  • Good for comparing all stores at once

Cons

  • Can clutter the page if many stores
  • Less "poster" feel
